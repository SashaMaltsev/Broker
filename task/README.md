# In-memory Queue Broker

HTTP-сервис на Go, который хранит сообщения в именованных FIFO-очередях в памяти.

## Запуск

Порт передается первым аргументом командной строки:

```bash
go run main.go 8080
```

Сервис начнет слушать `:8080`


## API

### Добавить сообщение

```http
PUT /{queue}?v={message}
```

Пример:

```bash
curl -i -XPUT 'http://127.0.0.1:8080/pet?v=cat'
```

Ответ:

- `200 OK` - сообщение принято;
- `400 Bad Request` - параметр `v` отсутствует.

Параметр `v=` с пустым значением считается корректным сообщением.

### Получить сообщение

```http
GET /{queue}
```

Пример:

```bash
curl -i 'http://127.0.0.1:8080/pet'
```

Ответ:

- `200 OK` и сообщение в теле ответа, если очередь не пуста;
- `404 Not Found`, если сообщения нет.

Сообщения выдаются в порядке FIFO.

### Получить сообщение с ожиданием

```http
GET /{queue}?timeout=N
```

`N` - время ожидания в секундах.

Пример:

```bash
curl -i 'http://127.0.0.1:8080/pet?timeout=5'
```

Поведение:

- если сообщение уже есть, оно возвращается сразу;
- если очередь пуста, запрос ждет новое сообщение до истечения timeout;
- если сообщение пришло вовремя, возвращается `200 OK`;
- если timeout истек, возвращается `404 Not Found`;
- если `timeout=0`, запрос не ждет;
- если `timeout` не является целым неотрицательным числом, возвращается `400 Bad Request`.

Ожидающие получатели обслуживаются в порядке поступления GET-запросов.

## Примеры проверки

```bash
curl -i -XPUT 'http://127.0.0.1:8080/pet?v=cat'
curl -i -XPUT 'http://127.0.0.1:8080/pet?v=dog'

curl -i 'http://127.0.0.1:8080/pet'
curl -i 'http://127.0.0.1:8080/pet'
curl -i 'http://127.0.0.1:8080/pet'
```

Ожидаемый результат:

- первые два запроса `PUT` вернут `200 OK`;
- первые два запроса `GET` вернут `cat` и `dog`;
- третий `GET` вернет `404 Not Found`.

Проверка ожидания:

```bash
curl -i 'http://127.0.0.1:8080/wait?timeout=5'
```

Пока запрос ждет, в другом терминале:

```bash
curl -i -XPUT 'http://127.0.0.1:8080/wait?v=hello'
```

Ожидаемый результат: ожидающий `GET` вернет `hello`, а `PUT` вернет `200 OK`.
